# RX5808 Channel OSD (drawtext overlay on the grabber stream) — Design Spec

**Date:** 2026-06-18
**Author:** andriy@netlife.com.ua
**Status:** Approved (pending written-spec review)

## 1. Context & Goal

The RX5808 grabber video is pushed to MediaMTX by `ffmpeg-stream.service` on the Pi
(`/dev/video0` 720×576 PAL → libx264 → RTSP `pi-01`). The operator wants the **current RX5808 channel
overlaid on the video** so the stream itself shows what's being received.

The Pi's `Rx5808Controller` already knows the current channel/mode (it tunes the RX5808 and publishes
`rxtune`). The overlay is rendered by ffmpeg's `drawtext` filter, which can read its text from a file
and reload it live. So: **the controller writes the current channel to a text file; the ffmpeg command
draws it.**

## 2. Confirmed decisions

| Decision | Value |
|---|---|
| Overlay text | `"{channel} · {freq} · {mode}"` → e.g. `A1 · 5865 · auto` (channel + freq + operating mode). |
| Position | **Top-right**, white text on a semi-transparent black box. |
| Text source | The controller writes the line to a file **atomically** (temp + `os.replace`) on every tune, so `drawtext reload=1` never reads a half-written file. |
| File path | `FPV_RX_OSD_FILE`, default **`/run/fpv/rx5808.txt`** (tmpfs; empty string disables). The controller `makedirs` its parent. |
| Render | A `drawtext` filter added to `ffmpeg-stream.service` (`-vf`), reading the file with `reload=1`. The unit is hand-managed on the Pi — edited in place; a reference copy ships in the repo. |
| Cadence | Updates with the hop cadence (~4 s) — the overlay follows channel/mode changes live. |

## 3. Architecture & data flow

```
Rx5808Controller.tune()  ──atomic write──▶  /run/fpv/rx5808.txt  ("A1 · 5865 · auto")
   (Pi scan service, root)                          │  reload=1
                                                     ▼
/dev/video0 (grabber) ─▶ ffmpeg -vf "drawtext=textfile=/run/fpv/rx5808.txt:reload=1:x=w-tw-10:y=10…"
                          ─▶ libx264 ─▶ RTSP pi-01 ─▶ MediaMTX ─▶ dashboard tile (overlay visible)
```

The controller's file write is best-effort and fully guarded (a write error never disturbs tuning).
The ffmpeg pipeline is otherwise unchanged — `drawtext` runs before the existing encode.

## 4. Components / deliverables

```
agent/scan/config.py             (change: + rx5808_osd_file + env FPV_RX_OSD_FILE)
agent/scan/rx5808_controller.py  (change: __init__ osd_file; tune() writes the OSD line atomically)
agent/scan/main.py               (change: pass cfg.rx5808_osd_file to the controller)
agent/scan/tests/test_rx5808_controller.py (change: OSD file written with the right content; disabled when empty)
agent/scan/tests/test_config.py  (change: rx5808_osd_file default/env)
systemd/ffmpeg-stream.service.example  (new: reference unit with the drawtext filter + ExecStartPre)
```

### 4.1 `config.py`
Add `rx5808_osd_file: str = "/run/fpv/rx5808.txt"` (env `FPV_RX_OSD_FILE`; empty disables the write).

### 4.2 `Rx5808Controller`
- `__init__` gains `osd_file=""` (passed from `main()`); store `self._osd_file`.
- A private `_write_osd(text)`: if `self._osd_file`, `os.makedirs(dirname, exist_ok=True)`, write
  `text` to `osd_file + ".tmp"`, then `os.replace(tmp, osd_file)`. Wrapped in try/except (log + continue).
- In `tune(name, freq, mode, target_freqs, ts)`, after the `set_frequency` call succeeds, call
  `self._write_osd(f"{name} · {int(round(freq))} · {mode}")`. (Skip the write if the tune raised.)

### 4.3 `main()`
Pass the path into the constructor: `Rx5808Controller(..., cfg.rx5808_settle_ms, osd_file=cfg.rx5808_osd_file)`.

### 4.4 `systemd/ffmpeg-stream.service.example`
A reference copy of the Pi's hand-managed unit, with:
- `RuntimeDirectory` / `ExecStartPre=/bin/sh -c 'mkdir -p /run/fpv; [ -f /run/fpv/rx5808.txt ] || echo RX5808 > /run/fpv/rx5808.txt'` so the file exists before ffmpeg starts (drawtext fails on a missing textfile).
- `-vf "drawtext=fontfile=/usr/share/fonts/truetype/dejavu/DejaVuSans-Bold.ttf:textfile=/run/fpv/rx5808.txt:reload=1:x=w-tw-10:y=10:fontsize=24:fontcolor=white:box=1:boxcolor=black@0.5:boxborderw=6"`
  inserted between `-i /dev/video0` and `-c:v libx264`.

## 5. Error handling & resilience

- `_write_osd` is guarded: a missing dir is created; any write/replace error is logged and swallowed —
  tuning and rxtune publishing are unaffected. Atomic `os.replace` guarantees `drawtext` reads a
  complete line.
- If `FPV_RX_OSD_FILE` is empty, the controller writes nothing (overlay simply shows the ExecStartPre
  placeholder, or the filter can be omitted).
- ffmpeg `drawtext` requires the textfile to exist at start; `ExecStartPre` guarantees it. A missing
  font file would fail ffmpeg — `fonts-dejavu-core` must be installed (deploy note).

## 6. Testing / verification

- **`test_rx5808_controller.py`:** with `osd_file=str(tmp_path/"osd.txt")`, `set_command("manual","A1")`
  + `run_once()` writes `"A1 · 5865 · manual"`; an empty `osd_file` writes nothing (file absent).
- **`test_config.py`:** `rx5808_osd_file` default `/run/fpv/rx5808.txt` + `FPV_RX_OSD_FILE` override.
- Run: `cd agent/scan && python -m pytest tests ../video/tests -q`.
- **Live (after deploy):** `cat /run/fpv/rx5808.txt` on the Pi shows the current channel line and
  updates as the RX5808 hops; the `pi-01` stream (dashboard tile / WHEP) shows the overlay top-right
  following the channel.

## 7. Out of scope

- The grabber/ffmpeg pipeline itself (only the `drawtext` filter + ExecStartPre are added) and ffmpeg
  autostart.
- Font installation (`fonts-dejavu-core`) — a one-time `apt` if absent.
- Overlaying anything beyond the RX5808 channel line; changing the controller modes or the MQTT contract.
